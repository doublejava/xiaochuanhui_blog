# kill 命令

## 命令简介
`kill` 命令用于终止指定PID的进程，通过发送信号来控制进程。

## 语法
```bash
kill [选项] PID...
```

## 常用信号
- `1` (SIGHUP)：挂起信号，重新加载配置
- `2` (SIGINT)：中断信号，相当于 Ctrl+C
- `9` (SIGKILL)：强制终止信号，无法被忽略
- `15` (SIGTERM)：终止信号，默认信号，允许进程清理
- `18` (SIGCONT)：继续信号
- `19` (SIGSTOP)：停止信号，暂停进程

## 常用选项
- `-l`：列出所有可用的信号
- `-s`：指定发送的信号
- `-9`：强制终止进程（相当于 `-s SIGKILL`）

## 示例
1. 发送默认信号终止进程：
   ```bash
   kill 1234
   ```

2. 强制终止进程：
   ```bash
   kill -9 1234
   ```
   或
   ```bash
   kill -s SIGKILL 1234
   ```

3. 重新加载进程配置：
   ```bash
   kill -1 1234
   ```

4. 列出所有信号：
   ```bash
   kill -l
   ```

5. 终止多个进程：
   ```bash
   kill 1234 5678 9012
   ```

## 注意事项
- 默认发送 SIGTERM (15) 信号，允许进程进行清理工作
- SIGKILL (9) 信号会强制终止进程，可能导致数据丢失
- 只有进程所有者或 root 用户可以终止进程
- 可以使用 `ps` 或 `pgrep` 命令查找进程PID
- 结合 `pkill` 命令可以更方便地终止进程，如 `pkill nginx`
